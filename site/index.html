An open research infrastructure project

One format for every trace of a decision.

OpenTrace is an open data standard and curated corpus for process-tracing data in decision research — so that eye movements, cursor paths, information acquisitions, and verbal protocols recorded in one lab can be read, reused, and re-analysed in any other.

01 Why

Rich data, locked in a thousand formats.

Process-tracing methods open a window onto how decisions are made, not just their outcomes. Decades of studies have recorded fixations, acquisitions, cursor trajectories, and verbal reports — yet almost every dataset lives in a lab-specific format shaped by one vendor's hardware, one experiment builder, and one analysis script.

The result: data that cannot move. Reuse, replication, cross-paradigm comparison, and cumulative meta-analysis all stall at the file format. OpenTrace exists to remove that barrier.

02 The standard

A common language for traces.

At its core, OpenTrace describes every study as the same three things: stimuli with defined areas of interest, trials linking participants to stimuli, and a stream of timestamped events — fixations, acquisitions, movements, utterances, choices. One schema, regardless of how the trace was recorded. 

// one trial, OpenTrace event stream
{
  "trial_id":  "S07-T012",
  "paradigm":  "eye_tracking",
  "stimulus":  "gamble_pair_03",
  "events": [
    { "t": 0.000, "type": "trial_start" },
    { "t": 0.184, "type": "fixation", "aoi": "A.outcome_1", "dur": 0.212 },
    { "t": 0.396, "type": "saccade",  "from": "A.outcome_1", "to": "B.outcome_1" },
    { "t": 2.741, "type": "choice",   "value": "B", "rt": 2.741 }
  ]
}

Format-agnostic

The schema is defined independently of storage. Serialise to CSV, JSON, or Parquet — the structure is the same.

Paradigm-general

Fixations, cell openings, cursor samples, and coded utterances are all events on a shared timeline with shared AOI references.

Validated

A machine-readable specification with reference validators, so a dataset either conforms or tells you exactly why it doesn't.

Documented by design

Required metadata — apparatus, sampling rate, preprocessing, task structure — travels with the data, not in a drawer.

03 The corpus

Not just a format — a library.

A standard is only useful if data actually exists in it. Alongside the specification, OpenTrace curates a growing, openly licensed corpus of process-tracing datasets converted to the common schema — spanning paradigms, tasks, and labs.

Every corpus entry ships with complete task and apparatus metadata, a provenance record pointing back to the original publication and raw files, and a license that permits reuse. The goal is a body of data large and consistent enough to support questions no single study can answer: cross-task regularities in information search, robust benchmarks for process models, and genuinely cumulative meta-analysis.

04 Roadmap

Where this is going.

  1. m1

    Schema draft

    A public v0.1 specification of the data model — entities, event taxonomy, metadata requirements — open for community comment.

  2. m2

    Reference converters & validators

    Open-source tooling that converts common vendor and experiment-builder outputs into OpenTrace, and validates conformance.

  3. m3

    Seed corpus

    An initial set of published datasets across eye-tracking, Mouselab, and mouse-tracing paradigms, fully converted and documented.

  4. m4

    Community governance

    A transparent process for evolving the specification and reviewing corpus submissions, owned by the research community.
